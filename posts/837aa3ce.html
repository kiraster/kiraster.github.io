Ubuntu publickey方式SSH连接与vscode远程调试

 Apr 27th 23  Python 5 minutes read (About 747 words)

Ubuntu服务端采用publickey方式验证客户端身份，客户端本地vscode连接至Ubuntu服务端进行python远程调试

环境描述：

  • vscode 1.77.3
  • VMware Station 15.0
  • Ubuntu 22.04.1
  • Python 3.10.6

本地用户密钥生成

生成安全的密钥

  • 根据需要是否修改保存的路径和文件名（可能会与之前生成的密钥同名）
  • 根据需要是否设置passphrase口令（如设置每次验证需要输入该密码）

上传pub文件至服务端

  • 将.pub公钥文件上传至ubuntu 22.04 ~/.ssh(/home/kir/.ssh)目录下

  • 追加.pub公钥文件至authorized_keys

  • 重启ssh服务

  • 本地连接测试

vscode设置

  1. 插件——搜索——安装 Remote - SSH 插件

  2. 点击左侧图标，点击REMOTE–设置，弹出的菜单中选择电脑的config文件路径

  3. 由于我本机还设置了github和gitlab的公钥，只需在config文件增加对应配置

  4. 选择ubuntu服务器连接

  5. 登陆后，在控制台手动创建目录，再左侧添加目录

    1. 创建目录

    2. 设置权限（为了方便，就设777）

  6. Python插件安装

  2. 安装第三方库–测试

  3. 虚拟环境–测试

最后

  • 总体来说能用，就是非root用户跑python有可能会遇到权限问题（可能需要sudo）
  • 为什么要搞远程python，因为发现在windows平台跑python时不时会遇到一些第三方库安装运行时出的一些不知道时什么不知道的问题
  • 欢迎“来电”来函探讨
